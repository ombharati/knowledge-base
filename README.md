# Knowledge Base
> *An evolutionary engineering scratchpad bridging high-level software abstractions with silicon realities.*

Welcome. My name is **Om Bhartai**, and I am a student of computer systems and low-level engineering. My technical focus lies at the intersection of software, networking & infrastructure enginnering. 

I built this space to document the journey of pushing high-level software constructs down to the metal—from register-level manipulations and VLSI concepts to the physical signals that govern our machines. Whether it is mapping asymmetric cryptographic handshakes like WebAuthn directly to how hardware TPMs and Secure Enclaves isolate private keys, this repository is where theory meets the unforgiving reality of hardware.

### Purpose of This Repository
This collection is not a static textbook, a sanitized portfolio, or a generic copy-paste library. It serves as a long-term, evolutionary engineering scratchpad. 

Its primary purpose is to capture the *operational friction* of building, breaking, and understanding complex systems. Real engineering is rarely clean, and this repository reflects the authentic, human-driven process of navigating architectural constraints, wrestling with the OS kernel, and ultimately forcing hardware and software to cooperate. 

### Learning Methodology & AI Integration
As an engineer with an advanced background in prompt engineering, my relationship with Artificial Intelligence is highly deliberate. **I do not use AI to blindly generate content, nor will you find artificial, sterile machine-symmetry in these notes.** 

Instead, I treat AI models as rigorous, deep-tech sparring partners. I leverage them to aggressively stress-test my architectural models, locate obscure cryptographic vulnerabilities, and dissect edge cases within the operating system kernel. AI is a tool used to augment my own cognitive heavy lifting, ensuring that the insights captured here are pressure-tested and deeply understood before they are ever written down.

### The Anatomy of This Workspace
To accurately reflect the realities of systems engineering, the documentation within this knowledge base is structured organically:

*   **Asymmetrical Analysis:** Cognitive effort is allocated where it is needed most. You will find exhaustive, granular deep dives into highly complex systems—such as Kerberos authentication flows, Linux bridge forwarding mechanics, and WebAuthn hardware isolation—contrasted with brief, utilitarian notes for simple concepts.
*   **Friction & Debugging Logs:** Engineering is defined by its failures. Rather than hiding mistakes, this repository actively preserves compiler errors, memory faults, and kernel panics. Documenting the exact friction points and the analytical steps taken to resolve them is treated as a first-class priority.
*   **Deconstructed Code Blocks:** Large, monolithic utilities are broken down into bite-sized, heavily commented C and Python snippets. Each block is clinically dissected to expose its underlying mechanics, memory management principles, and direct hardware interactions.
